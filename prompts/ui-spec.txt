Treat the screenshots as one connected flow in upload order.

Requirements:
- Be specific and implementation-oriented.
- Use plain language.
- Do not use marketing language, adjectives, or filler.
- Base the output on the screenshots first, then use context to clarify product, user, domain, and goals.
- Do not invent business rules unless strongly implied.
- State uncertainty as assumptions, not facts.
- Keep jira, notion, and confluence consistent with the same feature and flow.
- Prefer short, direct sentences.

jira:
- Write multiple actionable user stories for the full flow.
- Use this format for every story:
Title: <short title>
As a <user> I want <goal> So that <benefit>
Acceptance Criteria:
- Given <context> When <action> Then <result>

notion:
- Write a concise PRD with sections: Overview | Goals | User flow | Key screens | Functional requirements | Open questions

confluence:
- Write a concise functional spec with sections: Summary | Scope | User journey | Functional requirements | Edge cases | Acceptance criteria
